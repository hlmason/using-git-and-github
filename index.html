2. Configure Git

  • Go to your command line. For example:
  •  mc-macs-Mac-mini:~ mc-mac$  

  • Tell Git your name:
  •  git config --global user.name "Your Name"

  • Tell Git the email address you want associated with your 'commits':
  •  git config --global user.email "Your Email Address"

3. Set Up a GitHub Account

4a. Joining a Project as a Collaborator

Setting up

  • Navigate to the place on your computer where you want to store the project. For example:
  •  mc-macs-Mac-mini:htdocs mc-mac$  

  • Make sure you are logged into your GitHub account. Find the project info and clone the repository:
  •  git clone https://github.com/username/reponame

  • Add a 'remote':
  •  git remote add origin https://github.com/username/reponame.git

Getting Updated Files from and to GitHub

  • First, make sure your files are up-to-date with those on GitHub:
  •  git pull origin master

  • Check your 'commit' status:
  •  git status
  • Note: Do this as many times as you need throughout the commit process.

  • Add the paths of the files you are ready to commit:
  •  git add filepath

  • When you are finished adding the files for this commit, then you are ready to commit. Make a short note for your commit so that if you ever need, you have a general idea of what the commit was for (e.g. Added new feature, Revised content, etc.).
  •  git commit -m "Your Commit Message"

  • When you are finished commiting the files, then you are ready to send the updates to GitHub.
  •  git push origin master
